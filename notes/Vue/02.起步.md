# 3.`npm`常用命令

`node package manager`

- `npm init` 初始化项目。
- `npm init -y` 跳过向导，快速生成项目说明书。
- `npm install` 将`dependencies`中的依赖项全部安装回来。
- `npm install`  + 包名，只下载对应包名。
- `npm install --save` + 包名，下载并保存依赖项。即`package.json`中的`dependencies`依赖项。
- `npm uninstall` + 包名，仅删除包，依赖项依旧保存着。
- `npm unistall --save` + 包名，删除包的同时将依赖信息也删除。

# 4.关于`package.json`文件

我们建议每个项目的根目录下都要有一个`package.json`文件，用以描述项目，就像产品说明书一样。

这个文件可通过`npm init`的方式来自动初始化出来，通过问一句，答一句的形式生成项目说明书。其中`dependencies`选项，帮我们保存了项目所依赖的第三方包的信息。

**作用**：当`node_modules`文件丢失的时候，只要存在`package.json`文件，直接`npm install`就可以将项目所依赖的包全部都下载回来。

# 5.`nrm`

**作用**：提供最常用的`npm`包镜像地址，能够让我们快速的切换安装包时候的服务器地址。

**镜像**：包只存在于国外的`npm`服务器上，由于网络的原因，经常访问不到。这时我们在国内创建一个和官网完全一样的`npm`服务器，只不过数据都是从人家那边拿过来的，除此之外，使用方式完全一样。

[npm下载包的地址](registry.npmjs.org)，此网站位于国外，下载速度不友好。

`nrm`的出现，提高了装包的速度：

- `npm install nrm --global` 全局安装`nrm`。
- `nrm ls` 查看当前所有可用的镜像源地址以及当前正在使用的镜像源地址。

- `nrm use npm` 切换不同的镜像源地址。

==注意==：`nrm`只是提供了几个常用的下包地址，我们每次装包时所使用的装包工具依旧是`npm`。

# 6.`Vue`脚手架搭建项目

- `npm install @vue/cli --global` 
- `vue --version`
- `npm uninstall vue-cli --global` 卸载旧版本的脚手架
- `vue create` + 项目名
- 选工具，上下键进行切换，选择自定义，按空格键进行确定，回车。
- 选代码规范，选只是错误而已`with error prevention only`，回车。
- 选`Lint`特性，按照默认的，回车。
- 是否将每个文件都单独生成一个配置文件，还是直接生成一个`package.json`。
- 你会将上述选择当作一个习惯进行保存吗？
- 安装完成。
- `npm run serve`启动项目。

**tip**：方法（`methods`）和函数（`function`）之间有什么区别？

方法：定义在类里面

```js
function Person() {
    // ...定义在类里面的一般叫做方法
    // 方法一般都是和某一个实例对象相挂钩的
}
```

函数：暴露在外面（全局下）

```js
function callMe() {} // 直接写下全局下,暴露在全局下的叫做函数
```
